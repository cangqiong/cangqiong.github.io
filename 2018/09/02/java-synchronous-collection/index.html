Java的容器

Java的容器是存放多个对象的引用并且能够作为一个将其作为组进行管理的数据结构。Java的容器可以分成两大类：Collection与Map。Collection是管理一组允许重复的对象。Map是一组成对的键-值对象。基于Collection与Map接口及其实现类构成了Java的集合类。其中Collection接口实现了Iterable接口，这意味着Collection集合都可以采用迭代器的方式进行迭代访问。具体的继承结构如下图所示：

CopyOnWrite容器

Java还提供一种CopyOnWrite容器。顾名思义，当进行写操作的时候进行复制一份副本，修改完成后将副本作为容器对象替换原始对象，这其实本质是一种延时懒惰策略。

上面是CopyOnWriteArrayList的代码add方法的代码实现，可以看注释了解其实现。但与synchronized容器不同，同步实现是基于ReentrantLock重入锁进行手动加锁与释放锁。

CopyOnWrite容器容器适用于读多写少的场景，比如黑名单、关键词，一天更新一次那种。当写的次数比较多的时候，会产生多个容器副本，有内存溢出的风险，也可能造成垃圾回收的卡顿。

总结

Java容器在多线程开发中可能会出现线程同步的问题，可以采用自带线程安全的容器类；也可以采用Collections工具类的synchronized方法与自带的CopyOnWrite容器，构造同步的容器类；Java的同步容器底层实现基于synchronized或者Lock对象。
